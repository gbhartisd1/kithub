Notes for Backend Implementation:
  • Each build is an object in the database with fields: title, creator, photo, attributes (tags), ranking (average stars, # of reviews), bill of materials (array), description, and forum/comments (array or thread).
  • This static page demonstrates the frontend layout. Replace static data with dynamic content from the database.
  • Each build card should be clickable or have a “View & Discuss” link that leads to a detailed build page with full description and forum.
  • Ranking is an average of user-submitted star ratings. Reviews count is total ratings submitted.
  • Attributes are tags users can select when posting a build (e.g., fun, useful, creative, challenging).
  • Bill of materials should be an array/object in the backend; displayed as a bulleted list.
  • Forum is a thread per build, storing comments/replies. This can be a separate page or modal.
  • Images are uploaded and stored with the build object, with a URL to display here.
